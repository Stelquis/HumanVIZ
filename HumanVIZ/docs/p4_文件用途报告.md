## 文件一：`p4_opera_processor.py`

### 用途

**京剧原始 JSON → Story Ribbon 兼容格式的数据预处理管线。**

HumanVIZ 数据文件夹 `data/dataSet/` 下有 1,473 部京剧剧本的原始 JSON，其字段结构（`剧本名字`、`主要角色`、`情节`、`正文对话`）与现有故事丝带可视化管线所需的 gatsby-new 格式（`scenes[]` 含完整人物详情、情感评分、重要性排序）完全不同。本脚本完成这一转换。

### 核心功能

1. **场景分割**（`split_scenes`）
   - 正则匹配 `【第X场】` / `【第X场：标题】` / `【第X折】` 三种标记格式
   - 对无场景标记的剧本（约 21.5%），按角色退场点 `（X同下。）` 自动切分
   - 退场点不足时降级为按空行分段

2. **角色提取与验证**（`extract_characters_from_text`）
   - 行首 1-4 个中文字符 + `（表演类型）` 的严格格式匹配
   - 表演类型必须属于 40+ 种有效关键词（含"唱念做打哭笑"、西皮/二黄板式、昆腔/吹腔等）
   - 与剧本声明的"主要角色"列表交叉验证，弥补文本提取遗漏

3. **叙事指标自动计算**
   - **情感分析**（`compute_scene_sentiment`）：基于内置正/负面情感词典 + 表演标记权重（唱 0.8 / 做 0.7 / 打 1.0 / 哭 -0.7）混合打分
   - **场景重要性**（`compute_scene_importance`）：角色数量 × 行数占比 × 主角出场率的加权合成
   - **冲突度**：对立词（但/却/杀/战/争等）密度归一化

4. **行当分组映射**
   - 四大分组：生（老生/小生/武生/红生/外）、旦（青衣/花旦/武旦/老旦/刀马旦/彩旦）、净（正净/副净/武净/铜锤花脸/架子花脸）、丑（文丑/武丑）
   - 非标准行当降级为"其他"

5. **章节聚合**（`build_chapter_data`）
   - ≤4 场景 → 1 章；5-8 场景 → 2-4 章；9+ 场景 → 3-9 章
   - 自动计算章节级冲突度（情感方差）、重要性、角色共现网络

### 输入/输出

| 方向 | 格式 | 示例 |
|------|------|------|
| **输入** | 京剧原始 JSON（`正文对话` + `主要角色` + `情节`） | `01001001_空城计.json` |
| **输出** | gatsby-new 兼容 JSON（含 `scenes[]` `characters[]` `chapters[]` `locations[]`） | `opera-samples.json` |

### CLI 用法

```bash
python3 scripts/p4_opera_processor.py --samples       # 默认：处理 10 个代表性样本
python3 scripts/p4_opera_processor.py --all            # 批量处理全部 1,473 部
python3 scripts/p4_opera_processor.py -i file.json     # 单文件处理
python3 scripts/p4_opera_processor.py --list-samples   # 列出代表性样本
```

### 关键复用

- 输出格式完全对齐 `getAllData()` 的输入期望（`data.ts:799-953`），可直接喂入 `positionStore` → `getAllPositions()` 计算丝带路径

## 文件二：`p4_story_ribbon_core.ts`

### 用途

**泛用性故事丝带分析核心模块，对外部数据源提供统一 API。**

将现有的 `getAllData` / `getAllPositions` / `bezierCommand` / `svgPath` 等分散在 4 个文件中的函数封装为一条完整的处理流水线，使非 gatsby-new 数据源（如京剧剧本、其他文本叙事分析）可以直接调用。

### 核心 API

| 接口 | 类型 | 说明 |
|------|------|------|
| `normalizeRawInput(input)` | 函数 | 松散格式 `RawStoryInput` → gatsby-new 兼容 JSON |
| `analyzeStoryRibbons(input)` | 函数 | 完整流水线：标准化 → 数据结构化 → 位置计算 → 指标提取 |
| `createRibbonPath(top, bottom, adj)` | 函数 | 复用 `bezierCommand`/`svgPath` 生成单条丝带 SVG path |
| `extractFingerprint(result)` | 函数 | 从分析结果提取叙事结构指纹 |
| `computeDefaultScenePositions(n, w)` | 函数 | 独立模式下（无 store）计算场景默认坐标 |

### 类型定义（供外部使用）

- `RawStoryInput` / `RawScene` / `RawCharacter` — 宽松的输入格式，降低数据准备成本
- `RibbonAnalysisResult` — 完整的分析输出（含 `RibbonPositions`、`ribbonPaths`、`NarrativeMetrics`）
- `NarrativeMetrics` — 情感弧线、冲突变化、角色密度、节奏类型（渐进推进型/密集高潮型/长篇铺陈型/文武交替型）
- `StoryFingerprint` — 结构指纹（场景数/角色数/平均密度/情感波动/场景长度变异系数）

### 设计原则

1. **只包装，不修改**：所有核心逻辑来自现有模块，本文件仅做聚合与类型适配
2. **双向兼容**：同时支持原 gatsby-new 格式（通过 `getAllData` 直接解析）和新 `RawStoryInput` 格式（通过 `normalizeRawInput` 转换）
3. **无状态依赖**：核心函数不需要 Zustand store，可在任意上下文调用

### 关键复用

- `getAllData()`（`data.ts:799`）— 数据结构化
- `getAllPositions()`（`positions.ts:846`）— 角色坐标与路径计算
- `bezierCommand()` / `svgPath()`（`curve.ts:253/304`）— 贝塞尔曲线渲染
- `reverseSvgPath()`（`positions.ts:441`）— 底部路径反向闭合

## 文件三：`p4_opera_ribbon_viewer.tsx`

### 用途

**京剧故事丝带可视化 React 组件**，提供交互式叙事结构浏览界面。

### 组件结构

| 子组件 | 功能 |
|--------|------|
| `OperaSelector` | 顶部下拉选择器（多剧本切换）+ 指纹徽章（X场/X角色/节奏类型） |
| `MetricsPanel` | 叙事指标面板（节奏类型、情感波动、场景密度、均匀度、总行数） |
| `RibbonDefs` | SVG 渐变定义（每角色/每分段独立线性渐变） |
| `SceneBackgrounds` | 场景背景矩形（hover 高亮） |
| `RibbonLayer` | 核心丝带层：贝塞尔路径 + 角色标记圆点 + 名字标签 |
| `SceneLabels` | 场景底部标签（倾斜文本） |
| `CharacterLegend` | 角色图例（按行当分组，hover 联动） |

### 交互行为

- **角色 hover**：高亮该角色全部丝带路径，其他角色淡出至 opacity 0.15
- **场景 hover**：高亮该场景背景矩形
- **剧本切换**：下拉选择器切换后，全部丝带路径和指标重新计算
- **指标折叠**：按钮切换叙事指标面板的显示/隐藏

### 可复用性

- 组件接受 `Map<string, RawStoryInput>` 格式的多剧本数据，不依赖特定数据源
- 所有子组件均为独立导出，可被其他页面引用
- `getRoleGroupColor()` 辅助函数可在任何需要行当配色的场景下调用

### 关键复用

- 渐变色生成逻辑参考 `Defs.tsx`（线性渐变 + 分段透明度）
- 路径渲染逻辑参考 `MainPlot.tsx:289-307`（贝塞尔路径 + 渐变填充）
- 标记点计算复用 `normalizeMarkerSize()`（`helpers.ts:24`）
- 颜色生成复用 `getColor()` / `getGroupColor()`（`colors.ts:92/105`）
