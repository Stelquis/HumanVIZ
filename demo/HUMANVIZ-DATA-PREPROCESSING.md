# 2.1.2 数据解析流水线

做人文研究时，我们手上的数据往往来自不同的地方，格式也各不相同。有的可能是从数据库导出的 Excel 表格，有的是网上下载的 CSV 文件，有的是整理好的 JSON 数据，还有的就是普通的 TXT 文本文件。以故事丝带系统为例，我们处理的是文学文本数据，系统需要支持把这几种格式的文件都接进来：Excel 表格可能包含人物档案信息，CSV 可能是场景地点列表，JSON 可能是已经结构化的章节数据，而 TXT 就是最常见的纯文本小说内容。

数据接进来之后，系统会先根据文件名判断这是什么类型的数据。比如文件名里有"人物"两个字，系统就知道这是人物数据；有"地点"就是地点数据；有"场景"就是场景数据。这样后面查找和使用的时候更方便。

原始数据通常都比较乱，需要做一些清理工作才能用。首先是文字编码的问题，不同来源的文件编码方式可能不一样，有的用 UTF-8，有的用 GBK，如果处理不好就会出现乱码。系统会自动检测并统一转换成标准格式。另外，文本中的各种引号也需要统一，有的是中文引号""，有的是英文引号""，还有的是直角引号，系统会把它们都转换成统一格式。其次是格式规范化，TXT 文本文件需要按章节或段落进行分割，系统会以空行为分隔符，把大段文字切分成小段落，并且给每一行加上行号标记，这样做的好处是后面可以精确定位某句话在原文中的位置。Excel 和 CSV 表格中的数据也需要处理，比如去除重复的空格、统一换行符、处理缺失值等。最后是元信息记录，系统会记下这个文件叫什么名字、有多少条数据、什么时候导入的，这些信息对后续管理和追溯很有用。

清洗好的数据还需要进一步加工，提取出我们关心的"实体"和它们的属性。在故事丝带系统中，实体就是人物、地点、场景这些研究对象。每条原始记录会被转换成标准格式的实体对象，包含几个基本信息：ID 编号由系统自动生成，名称通常提取自记录的第一个文本列作为显示标识，类型字段根据数据源类别自动赋值，以及详细的属性信息。属性信息采用灵活的方式存储，因为不同数据源的字段差异很大，人物数据可能有"出生年份""籍贯""身份"这些字段，地点数据可能有"经纬度""朝代""描述"这些字段。系统使用 JSON 格式来保存所有这些属性，既保留了原始数据的完整性，又方便后续查询。如果是文本类数据，系统还会进一步分析，提取出人物之间的关系，比如文本中出现了"林黛玉和贾宝玉"这样的描述，系统可以识别出这两个人物之间存在某种关联，把这种关系也单独记录下来。

当数据量很大时，逐条插入数据库会非常慢，系统采用了批量导入的方式来提升效率。具体来说，导入一个文件时，系统会先创建一条数据集记录说明这是什么数据，然后把文件里的所有实体记录打包，一次性批量插入数据库，这样做比一条条插入要快几十倍。如果同时要导入多个文件，系统还支持并发处理，多个文件的解析和入库操作可以同时进行，充分利用计算机的多核能力，当然并发数量会控制在合理范围内，避免给数据库造成太大压力。导入过程中如果遇到错误，比如某条记录格式有问题，系统会跳过这条记录继续处理后面的数据，而不会让整个导入任务失败，错误信息会被记录下来方便后续查看和修复。

数据导入完成后还需要做一些检查和处理确保数据质量。首先是数据完整性检查，系统会比对导入前后的记录数量看看有没有丢数据，必填的字段如果为空会被标记出来提醒处理。其次是去重处理，同一个实体在不同数据源中可能有不同的写法，比如"贾宝玉"可能被写成"宝玉""宝二爷""怡红公子"等，系统通过相似度匹配来发现这些可能是同一实体的记录并提供合并功能，合并时会整合所有信息确保数据不丢失。最后是索引优化，为了让查询更快，系统会在常用的查询字段上建立索引，比如按类别查找、按朝代查找等，这样即使数据量很大检索速度也能保持很快。

处理好的数据需要通过接口对外提供服务让前端界面能够方便地使用。系统提供了一系列数据服务：可以查看所有数据集的列表，可以查看某个数据集的详细内容，可以用关键词搜索数据，可以把数据导出成 JSON 或 CSV 格式。管理后台提供了可视化的操作界面，在仪表盘页面上可以看到当前有哪些数据集、总共有多少条记录，在数据管理页面可以浏览具体的数据内容、预览数据详情，在导入导出页面可以通过拖拽的方式上传新数据文件或者把数据下载到本地，技术人员还可以使用 SQL 查询器直接写查询语句来查找数据。

人文数据不是一成不变的，经常会有新增或修改的需求，系统支持增量更新不必每次都全量重新导入。当导入一个已经存在的数据集时，系统会提供几种策略：可以选择跳过已存在的数据只导入新增的，可以选择用新数据完全覆盖旧数据，也可以选择合并新旧数据只更新有变化的部分。每次导入操作都会生成一条日志记录，包括文件名、导入时间、成功多少条、失败多少条等信息，这些日志方便数据管理员追溯数据的变化历史，如果某次导入出了问题可以追溯到具体是哪一批次，必要时可以回滚到之前的状态。

经过这一整套流程的处理，原本杂乱无章的原始数据就变成了结构清晰、质量可靠、易于使用的标准化数据，可以为前端的故事可视化、人物关系分析等功能提供坚实的数据支撑。
