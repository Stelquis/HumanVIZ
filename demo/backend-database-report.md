# 后端框架搭建与数据库链接设计

## 3.1 后端框架搭建

### 3.1.1 系统核心功能

后端服务作为人文数据可视化系统的数据中枢，提供了数据管理和基础服务支撑两大核心功能。在数据管理方面，系统支持 Excel、CSV、JSON、TXT 四种格式的数据导入，用户通过网页端的拖拽上传区域即可完成数据入库。每条导入的数据会被自动解析并存储到数据库中，同时系统会根据文件名中的关键词自动识别数据类别，如包含"人物"的文件会被归入人物类数据集。

在基础服务支撑方面，后端提供系统健康检查接口，用于监控服务运行状态；提供数据集列表查询和详情获取接口；支持分页预览数据集内容；提供数据导出功能，可将数据库中的数据导出为 JSON 或 CSV 格式。后端还集成了大语言模型服务的连接能力，为后续可能的智能化功能扩展预留了接口。

### 3.1.2 管理后台页面功能

系统管理后台提供了八个功能页面，通过统一的侧边栏导航进行切换。

仪表盘页面是系统的首页，展示三类统计信息：第一行显示当前可用的数据集数量、配置的 LLM 提供商名称和当前使用的模型名称；第二行展示数据库的基本状态，包括数据库类型、表数量、总记录数和数据库文件大小；第三行展示系统版本信息、FastAPI 服务运行状态和 LLM 服务可用状态。所有数据实时从数据库和配置中读取，刷新页面即可获取最新状态。

数据预览页面提供数据集的分页浏览和筛选功能。页面顶部是数据集选择器，选择后下方显示筛选条件区域，用户可输入关键词进行搜索。数据以表格形式展示，默认每页显示 20 条记录，支持翻页浏览。表格列根据数据内容动态生成，可展示实体的 ID、名称以及各项属性字段。

### 3.1.3 服务启动与部署

后端服务基于 Python 的 FastAPI 框架构建，采用 Uvicorn 作为 ASGI 服务器运行。系统支持从环境变量读取配置，包括服务监听的端口、是否开启热重载调试模式等。启动时会自动检查 LLM 服务配置，如果配置正确则初始化模型连接，否则以降级模式运行仅保留数据管理功能。

服务启动后会输出访问地址信息，包括管理后台入口、API 文档地址和健康检查端点。系统会自动初始化数据库，创建所需的五张数据表，并扫描已存在的数据集文件建立索引。整个启动过程无需人工干预，执行启动脚本即可自动完成所有初始化工作。

## 3.2 数据库链接设计

### 3.2.1 数据库选型与 Schema 设计

系统选用 SQLite 作为数据库解决方案。SQLite 是嵌入式数据库，整个数据库存储在一个单一文件中，无需独立的数据库服务进程，非常适合中小规模的数据管理需求。对于人文数据分析场景，数据量通常在几万到几十万条记录之间，SQLite 完全能够胜任。同时单文件存储的特性使得数据备份和迁移变得非常简单，只需复制数据库文件即可。

数据库采用五表结构设计。datasets 表存储数据集的基本信息，包括名称、类别、朝代、记录数量等元数据。entities 表存储具体的实体数据，如人物、场景、事件等，每条记录包含所属数据集、实体名称、类型、内容和扩展属性。field_defs 表记录每个数据集包含的字段定义，用于动态生成数据预览页面的表格列。relations 表用于存储实体之间的关系，如人物之间的关联。import_logs 表记录数据导入的操作日志，包括文件名、导入状态和创建时间。

表之间通过外键建立关联。entities 表通过 dataset_id 与 datasets 表关联，表示每个实体属于哪个数据集。field_defs 表同样通过 dataset_id 与 datasets 表关联。relations 表通过 source_entity_id 和 target_entity_id 与 entities 表建立双向关联。当删除某个数据集时，关联的实体、字段定义和关系数据会自动级联删除，确保数据一致性。

### 3.2.2 数据访问与系统验证

数据访问采用原生 SQL 方式，通过上下文管理器管理数据库连接。系统使用 Python 的 sqlite3 模块建立连接，查询时启用行工厂模式将结果转换为字典格式便于处理。对于批量数据插入操作，使用 executemany 方法一次性提交多条记录，相比逐条插入可提升数十倍的性能。

系统安全性方面，SQL 查询器实施了严格的安全策略。只允许执行以 SELECT 开头的查询语句，自动检测并拦截包含 INSERT、UPDATE、DELETE、DROP 等危险关键字的输入，防止误操作或恶意攻击导致数据损坏。

系统联通性通过多种方式验证。健康检查接口返回服务状态、版本信息和 LLM 配置，可用于监控服务是否正常运行。数据导入导出功能经过多格式测试，支持包含中文内容的文件正确处理。SQL 查询器可执行各类 SELECT 查询并正确返回结果。所有功能均已在实际环境中验证可用，系统运行稳定。
