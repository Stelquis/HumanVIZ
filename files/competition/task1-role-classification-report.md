# ChinaVis 2026 赛道1-I
# 任务一《戏曲角色行当推断与演化分析》设计流程指导报告

## 一、任务目标解析

任务一的核心目标并非简单的"角色分类"，而是：
基于戏曲剧本中的人物信息、行为描写与表演提示，构建角色语义体系，完成行当推断，并进一步分析角色特征与行当之间的对应关系及其历史演化规律。

该任务本质上融合了：

- 自然语言处理（NLP）
- 数字人文（Digital Humanities）
- 戏曲角色研究
- 可解释分类
- 可视分析

因此，最终成果应体现：

| 层次 | 内容 |
|------|------|
| 数据层 | 角色结构化抽取 |
| 算法层 | 行当推断 |
| 分析层 | 特征-行当关联 |
| 历史层 | 行当演化规律 |
| 可视层 | 交互式可视分析 |

## 二、整体研究框架

建议整体流程如下：

```
剧本文本
  ↓
角色信息抽取
  ↓
角色特征结构化
  ↓
行当推断模型
  ↓
角色-行当关系分析
  ↓
历史时期演化分析
  ↓
可视分析与交互展示
```

## 三、数据预处理阶段

### 3.1 剧本文本解析

首先需要对原始剧本进行标准化处理。

建议提取：

| 数据对象 | 内容 |
|----------|------|
| 剧目名称 | 戏曲名称 |
| 创作年代 | 明清/民国/现代 |
| 角色名称 | 人物名 |
| 台词 | 人物对白 |
| 舞台提示 | 唱、念、做、打 |
| 出场顺序 | 剧情阶段 |
| 场次信息 | 幕、折、回 |

### 3.2 数据清洗

| 清洗任务 | 说明 | 示例 |
|----------|------|------|
| 去除特殊符号 | 清理空白字符、无意义标点、OCR错误 | — |
| 统一角色名称 | 建立角色别名映射表 | 包公 = 包拯 = 包龙图 |
| 标准化时间信息 | 将朝代描述统一为标准标签 | 清代→Qing，民国初年→Republican，当代→Modern |

## 四、角色特征建模

该阶段是整个任务的核心基础。

### 4.1 建立角色特征Schema

建议建立如下结构：

```
角色
├── 性别
├── 年龄
├── 身份
├── 性格
├── 情绪
├── 表演形式
├── 行为特征
├── 语言风格
├── 社会关系
└── 行当标签
```

### 4.2 特征抽取设计

特征抽取覆盖三个维度，采用"正则规则 + 关键词词典 + NER识别"联合完成：

| 维度 | 特征项 | 示例 | 方法 |
|------|--------|------|------|
| **基础属性** | 性别、年龄、身份、社会地位 | 男/女、少年/老年、官员/书生 | 正则 + 关键词词典 + NER |
| **性格特征** | 忠义、狡诈、泼辣、温婉等 | 忠臣/报国、阴险/奸诈 | 情感词典、TF-IDF关键词 |
| **表演形式** | 唱（情感表达）、念（台词主导）、做（身段动作）、打（武戏行为） | 构建角色→四功模式向量 | 剧本标注统计 |

表演模式向量示例：

| 角色 | 唱 | 念 | 做 | 打 |
|------|----|----|----|----|
| 包公 | 高 | 高 | 中 | 低 |
| 武将 | 中 | 中 | 高 | 高 |

## 五、行当推断模型设计

### 5.1 行当体系定义

建议建立标准行当树：

```
生
├── 老生
├── 小生
├── 武生
旦
├── 青衣
├── 花旦
├── 武旦
净
丑
```

### 5.2 推断策略

不建议纯黑盒分类。采用 **"规则知识库 + LLM 语义推断"双轨融合方案**：规则层提供可解释的戏曲学依据，LLM 层处理规则难以覆盖的模糊与边界情况，两者互补输出概率化推断结果。

### 5.3 第一阶段：规则推断

建立规则知识库：

| 规则 | 推断结果 |
|------|----------|
| 男性+老年+忠义 | → 老生 |
| 女性+活泼+年轻 | → 花旦 |
| 武力+打戏 | → 武生 |

优点：可解释性强，符合戏曲规律。

### 5.4 第二阶段：LLM 语义推断

利用大语言模型（如 GLM-4）作为"京剧行当分析专家"，基于剧本上下文进行角色行当推断。输入包括剧目信息、角色对白、已知行当标注，输出为每个角色的验证行当、置信度及推断依据。

该方案的优势在于：模型能综合理解角色的语言风格、行为描写与戏曲语境，处理规则难以覆盖的行当模糊与跨行当融合现象。

### 5.5 第三阶段：概率融合

将规则推断与 LLM 推断的结果进行融合，输出多行当概率分布而非单一标签：

```
角色A：
  老生 72%
  武生 18%
  净   10%
```

实际系统中通过交互式规则推断面板（RuleInferencePanel）实现：用户选择性别、年龄、武力、性格等属性后，系统动态匹配规则库并按置信度排序展示推断结果，使推断过程透明可解释。

## 六、角色-行当关系分析

该部分是主要 insight 输出阶段。

### 6.1 高频特征分析

| 行当 | 高频语义 |
|------|----------|
| 老生 | 忠、朝廷、老夫 |
| 花旦 | 娇羞、小姐 |
| 净 | 怒、大喝 |

### 6.2 特征贡献分析

综合运用 SHAP（特征解释）、PMI（特征关联）、TF-IDF（核心词发现）三种方法，量化各特征对行当推断的贡献度。

### 6.3 角色聚类分析

利用 UMAP / t-SNE 观察行当语义空间分布。

例如：老生角色聚集、花旦角色聚集、净角语义独立。

## 七、历史演化分析

这是整个任务的高价值部分。

### 7.1 时间维度分析

建立 年代×行当 二维矩阵，观察：
- 行当占比变化
- 角色复杂度变化
- 行当融合趋势

### 7.2 行当语义漂移

研究同一行当在不同年代的语义变化。

青衣示例：

| 时期 | 特征 |
|------|------|
| 清代 | 贞烈、端庄 |
| 现代 | 独立、情感丰富 |

推荐：Word2Vec Temporal Embedding、embedding drift。

## 八、可视化设计方案

核心图表选型及用途：

| 图表 | 用途 | 说明 |
|------|------|------|
| **Sankey Diagram**（重点） | 特征→行当关联 | 忠义→老生、滑稽→丑、武艺→武生，最适合展示行当推断逻辑 |
| **时间热力图** | 年代×行当演化 | 横轴年代、纵轴行当、颜色频率，用于行当演化分析 |
| **UMAP Projection** | 角色语义聚类 | 展示不同行当的语义空间分布 |
| **Radar Chart** | 行当人物画像 | 多维度刻画行当特征（如老生：忠义高、武力中、情感低、滑稽低） |
| **River Chart** | 行当变化趋势 | 叠面积图展示行当占比随时代的变化 |
| **Chord Diagram** | 特征共现关系 | 展示角色特征间的共现强度 |

## 九、交互系统设计建议

如果进入系统展示阶段，建议多视图联动。

**交互逻辑**：点击某角色 → 联动显示：
- 行当概率
- 特征词
- 历史时期
- 相似角色

**推荐布局**：

| 区域 | 内容 |
|------|------|
| 左侧 | 角色列表 |
| 中间 | 关系与聚类图 |
| 右侧 | 角色画像+行当推断结果 |
| 底部 | 时间演化趋势 |

## 十、论文/答卷中的创新点包装

建议写法：

- **创新点1**：提出融合角色属性、语言风格与戏曲表演提示的多维角色建模方法
- **创新点2**：构建规则知识库与 LLM 语义推断融合的可解释行当推断框架
- **创新点3**：提出跨时代戏曲角色-行当演化分析方法
- **创新点4**：设计面向戏曲数字人文研究的角色语义可视分析系统

## 十一、推荐技术栈

- **数据处理**：pandas（数据处理）、jieba/pkuseg（分词）、HanLP（NLP）、regex（规则抽取）
- **模型层**：BGE（语义embedding）、LightGBM（分类）、SHAP（可解释性）、HDBSCAN（聚类）
- **可视化**：D3.js（交互可视化）、ECharts（图表渲染）、React + TypeScript（前端框架）

## 十二、系统实现与成果展示

最终不要呈现为"一个分类器"，而应呈现为 **"戏曲角色语义与行当演化可视分析系统"**。核心强调：**可解释 · 可分析 · 可探索 · 有人文洞察**。

实际系统的关键可视化组件：

| 组件 | 实现方式 | 功能 |
|------|----------|------|
| **行当体系双环图** | ECharts 双层环形图 | 内环展示生/旦/净/丑四大类占比，外环展开11个子类，点击可查看详细数据 |
| **特征→行当 Sankey 图** | D3.js Sankey 布局 | 29个角色特征维度流向11个行当类型，直观展示特征与行当的关联强度 |
| **表演四功极坐标图** | ECharts 极坐标柱状图 | 以唱/念/做/打为四轴，展示8个典型角色的表演模式对比，支持极坐标/直角坐标切换 |
| **历史演化趋势图** | ECharts 多系列面积图 | 横轴8个历史时期，纵轴各行当子类占比变化，附关键发现标注 |
| **交互式规则推断面板** | React 组件 | 用户选择性别、年龄、武力、性格等属性，系统实时匹配11条推断规则并按置信度排序 |
| **数据集概览** | ECharts 水平柱状图 | 展示13个来源收录集及对应剧目数量分布 |

系统采用前后端分离架构：前端 React + TypeScript + ECharts + D3.js，后端 Python FastAPI + LangChain，支持 LLM 驱动的角色行当验证与剧目分类。
