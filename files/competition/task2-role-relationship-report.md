# ChinaVis 2026 赛道1-I
# 任务二《角色关系网络与剧目类型分析》设计流程指导报告

## 一、任务目标解析

任务二的核心目标是：基于全量京剧剧本的场景级角色共现数据，构建角色关系网络，量化网络拓扑结构指标，并比较不同剧目类型之间是否存在可区分的"关系结构指纹"。

该任务本质上融合了：

- 社会网络分析（SNA）
- 计量戏剧学（Computational Dramaturgy）
- 统计检验与假设验证
- 高维数据降维与可视分析

因此，最终成果应体现：

| 层次 | 内容 |
|------|------|
| 数据层 | 场景级角色共现网络批量提取 |
| 算法层 | 多维网络结构指标计算 |
| 分析层 | 剧目类型间网络拓扑差异的统计检验 |
| 洞察层 | 七种剧目类型的"关系结构指纹"画像 |
| 可视层 | 力导向图、PCA散点图、热力图等多视图交互系统 |

## 二、整体研究框架

建议整体流程如下：

```
剧本文本（1473本）
  ↓
场景切分与角色共现提取
  ↓
构建角色共现网络（nodes + edges）
  ↓
计算 8 项网络结构指标
  ↓
统计检验：ANOVA + Kruskal-Wallis + Tukey HSD
  ↓
剧目类型×网络结构差异分析
  ↓
降维与代表性网络提取
  ↓
可视分析与交互展示
```

## 三、数据预处理阶段

### 3.1 场景级角色共现提取

首先需要对全量 1473 本剧本进行场景级角色共现关系的批量提取。

核心方法为纯正则方案（无 LLM 依赖），保证处理效率与可复现性：

| 步骤 | 方法 | 说明 |
|------|------|------|
| 场景切分 | `re.split(r'【[^】]*(?:场\|折\|幕\|本\|出)[^】]*】', dialogue)` | 自适应多种场景标记格式 |
| 角色提取 | `re.findall(r'^([一-龥]{2,4})\s+（', scene_text)` | 匹配"角色名 （"格式的台词行 |
| 共现边构建 | 同场任意两角色间建立无向边 | 权重 = 共现场次数 |

### 3.2 网络构建

对每本剧本生成一个角色共现网络，包含：

| 数据对象 | 内容 |
|----------|------|
| 节点（nodes） | 角色名称、度中心性、出场场景数 |
| 边（edges） | 源角色、目标角色、共现权重 |
| 网络概要 | 总场景数、总角色数、总边数、网络密度、最大边权重 |

### 3.3 剧目类型映射

将全量 1473 本剧本映射至 7 种剧目类型：

| 剧目类型 | 剧本数量 | 典型特征 |
|----------|----------|----------|
| 历史戏 | 776 | 阵营对立、群像结构 |
| 家庭戏 | 223 | 家族内部、紧密团块 |
| 侠义戏 | 126 | 英雄单核、江湖桥接 |
| 爱情戏 | 116 | 才子佳人、双核链式 |
| 神话戏 | 115 | 天界-凡间-地府多层 |
| 公案戏 | 100 | 法官居中、辐射审案 |
| 技法展示戏 | 17 | 极简网络、独角或对戏 |

## 四、网络结构指标设计

该阶段是整个任务的核心分析基础。

### 4.1 指标体系定义

建议建立 8 项结构指标体系，从不同维度刻画网络拓扑特征：

```
网络结构指标
├── 密度（density）—— 连接紧密程度
├── 中心性偏离度（centralization）—— 是否存在超级枢纽
├── 聚类系数（clustering）—— 邻居间互连程度
├── 模块度（modularity）—— 社区分化程度
├── 有效直径（diameter）—— 网络跨度
├── 度分布熵（degree_entropy）—— 角色重要性均匀度
├── 桥接节点比（bridge_ratio）—— 跨社区枢纽占比
└── Top-2集中度（top2_concentration）—— 核心角色聚焦程度
```

### 4.2 指标计算设计

各指标的计算方法及其对剧目类型的区分力：

| 指标 | 计算方法 | 预期区分力 |
|------|----------|------------|
| **密度** | 实际边数 / 最大可能边数 | 区分家庭戏(高) vs 侠义戏(低) |
| **中心性偏离度** | max_degree / mean_degree | 区分"单核" vs "多核"结构 |
| **聚类系数** | 邻居之间也相连的平均比例 | 区分公案戏(高) vs 爱情戏(低) |
| **模块度** | 标签传播社区检测的 Q 值 | 区分历史戏(高) vs 技法展示(低) |
| **有效直径** | Floyd-Warshall 90%百分位最短路径 | 区分大型戏 vs 小型戏 |
| **度分布熵** | Shannon 熵归一化 | 区分家庭戏(高) vs 爱情戏(低) |
| **桥接节点比** | 跨社区枢纽节点占比 | 区分侠义戏(高) vs 技法展示(低) |
| **Top-2集中度** | 最强两边的权重占比 | 区分技法展示(高) vs 历史戏(低) |

### 4.3 统计检验设计

不建议仅凭均值比较下结论。采用 **"描述统计 + 参数检验 + 非参数检验 + 事后比较"多重验证方案**：

| 检验方法 | 用途 | 说明 |
|----------|------|------|
| ANOVA | 参数检验 | 检验 7 种类型间指标均值是否存在显著差异 |
| Kruskal-Wallis | 非参数检验 | 不依赖正态分布假设的稳健检验 |
| Tukey HSD | 事后比较 | 两两类型间的具体差异定位 |

该方案的优势在于：多重检验交叉验证，避免单一方法的假阳性风险，确保结论的统计学可信度。

## 五、关键发现与类型画像

### 5.1 统计显著性

全部 8 项指标在 7 种类型间均达到 **p < 0.001** 显著水平，验证了"不同类型京剧对应不同关系结构指纹"的核心假设。

### 5.2 部分假设修正

实际数据推翻了部分初始假设，体现了数据驱动分析的价值：

| 初始假设 | 实际发现 | 修正解读 |
|----------|----------|----------|
| 公案戏应为星形网络 | 聚类系数最高(0.855) | 公案戏是"密集核心+辐射散边"，衙役/官吏形成紧密团块 |
| 爱情戏应有高聚类 | 聚类系数最低(0.744) | 才子佳人模式中配角间缺乏横向联系 |

### 5.3 修正后的七类网络画像

| 剧目类型 | 网络结构画像 | 核心指标特征 |
|----------|------------|------------|
| **公案戏** | 密集核心（衙役/官吏团）+ 辐射散边（当事人/证人） | 聚类系数最高 |
| **家庭戏** | 扁平团块，角色权重最均匀 | 度分布熵最高 |
| **侠义戏** | 英雄单核 + 多社区桥接 + 低密度 | 中心性偏离度最高 |
| **历史戏** | 群像散焦 + 模块化阵营 | Top-2集中度最低 |
| **爱情戏** | 双核链式，中庸各项指标 | 各项指标居中 |
| **神话戏** | 中等规模，多层世界偶联 | 各项指标居中 |
| **技法展示戏** | 极简网络，2-4人 | 密度极高、角色数最少 |

### 5.4 枢纽角色分析

提取每种剧目类型中度中心性 Top-3 的枢纽角色，揭示不同类型的核心人物模式：

- **历史戏**：帝王将相（如刘备、曹操、诸葛亮），群像中多极分布
- **家庭戏**：家长型角色（如父亲、母亲），权重均匀无绝对主导
- **侠义戏**：英雄主角（如武松、林冲），单核辐射结构
- **爱情戏**：才子佳人双核（如崔莺莺、张生），配角围绕双核展开

## 六、PCA 结构空间分析

该部分将 1473 本剧本的 8 维网络指标降维至二维空间，观察类型聚类模式。

### 6.1 降维方法

采用主成分分析（PCA）进行线性降维：

| 主成分 | 方差解释率 | 含义 |
|--------|-----------|------|
| PC1 | 48.7% | 网络规模与复杂度 |
| PC2 | 18.2% | 网络集中度与结构模式 |

### 6.2 结构空间分布

对 500 本采样剧本进行 PCA 投影，颜色标记剧目类型，标注 7 个类型质心。预期观察：

- **历史戏**与**侠义戏**在 PC1 轴上分离（规模差异）
- **家庭戏**与**爱情戏**在 PC2 轴上分离（集中度差异）
- **技法展示戏**独立于主群之外（极简结构）

## 七、可视化设计方案

核心图表选型及用途：

| 图表 | 用途 | 说明 |
|------|------|------|
| **力导向网络图**（重点） | 代表性网络展示 | 节点=角色(大小=度中心性)，边=共现(粗细=权重)，按类型切换 |
| **PCA 散点图** | 结构空间探索 | 500 点采样投影，颜色=类型，hover 显示剧本信息 |

| **枢纽角色排名** | 核心人物发现 | 每类型 Top-3 枢纽角色 + 度中心性数值 |

## 八、交互系统设计建议

采用三面板布局实现多视图联动。

**布局设计**：

| 区域 | 内容 |
|------|------|
| 左侧 | 剧目类型列表（7种类型 + 剧本数量）+ 网络概览统计 + 关键发现 |
| 中间 | 主视图切换区（力导向图 / PCA散点图 / 指标热力图） |
| 右侧 | 结构洞察面板（每类型最高指标 + 类型特征速查） |

**交互逻辑**：

- 点击左侧剧目类型 → 中间主视图切换至该类型的代表性网络
- 切换主视图模式 → 在力导向图、PCA散点图、热力图间自由切换
- 右侧抽屉面板 → 关系网络画廊 / 结构指标对比 / PCA结构空间 / 枢纽角色排名 四选一

**推荐设计语言**：

采用与任务一统一的"燕京清晖"主题——木色/卡其/竹青/纸白为主色调，玻璃态面板（双层渐变+半透明层+圆角+阴影），PT Serif / Noto Serif SC 字体，14-16px 圆角面板，hover 浮动 + active 缩放的交互反馈。

## 十、推荐技术栈

- **数据处理**：Python（批量正则提取）、NumPy（数值计算）、pandas（结构化分析）

## 十一、系统实现与成果展示

最终呈现为 **"戏曲角色关系网络与剧目类型可视分析系统"**。核心强调：**可比较 · 可探索 · 有结构洞察 · 有统计支撑**。

实际系统的关键可视化组件：

| 组件 | 实现方式 | 功能 |
|------|----------|------|
| **力导向网络图** | ECharts graph layout | 展示每种类型的代表性角色共现网络，节点按行当着色，支持拖拽与缩放 |
| **PCA 结构空间散点图** | ECharts scatter | 500 本采样的 PCA 投影，7 类型质心标注，hover 显示剧本详情 |
| **指标热力图** | ECharts heatmap | 7类型×7指标归一化矩阵，直观呈现每类型的结构指纹 |
| **分组柱状图** | ECharts bar | 密度/中心性/聚类系数/模块度四指标的跨类型并排对比 |
| **关系网络画廊** | React 卡片组件 | 7类型×2本代表性网络的缩略图浏览 |
| **枢纽角色排名** | React 列表组件 | 每类型 Top-3 枢纽角色及其中心性指标 |

系统采用前后端分离架构：前端 React + TypeScript + ECharts，数据预处理 Python 脚本批量生成静态 JSON，支持 1473 本剧本的全量网络分析与交互探索。

## 十二、与任务一的协同设计

任务二与任务一在数据管道和设计语言上形成有机协同：

| 协同维度 | 任务一（角色行当） | 任务二（角色关系） |
|----------|-------------------|-------------------|
| 核心对象 | 单个角色的属性与行当 | 角色间的共现关系 |
| 分析粒度 | 角色级 | 网络级 |
| 可视化重心 | Sankey流图（特征→行当） | 力导向图（关系网络） |
| 分类对比 | 雷达图（行当画像） | 热力图（类型×指标） |
| 高维探索 | UMAP聚类 | PCA结构空间 |
| 设计语言 | "燕京清晖"主题 | 统一继承 |
| 数据管道 | 角色特征提取 | 场景共现提取（共享正则） |

两者共同构成从"角色个体分析"到"角色关系分析"的完整戏曲数字人文研究链路。
