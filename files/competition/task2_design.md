# 任务二：角色关系网络与剧目类型分析 — 完整设计方案

> 基于全部 1473 本京剧剧本的场景级角色共现提取，构建关系网络，比较 7 种剧目类型的网络结构差异

---

## 一、核心分析问题

不同类型京剧（历史戏、家庭戏、公案戏等）在**角色关系网络的拓扑结构**上存在什么本质差异？不同剧目类型是否对应不同的"关系结构指纹"？

### 可验证假设

| 剧目类型 | 预测网络拓扑 | 预测结构特征 | 戏剧逻辑 |
|---------|------------|------------|---------|
| **公案戏** | 星形/辐射状 | 低密度、高中心性偏离度、1个超级枢纽 | "包公审案" — 法官在圆心，当事人、证人、衙役在圆周 |
| **家庭戏** | 紧密团块状 | 高密度、高聚类系数、多核心共享 | "家族内部" — 3-5个家庭成员形成闭合小团体 |
| **历史戏** | 模块化/哑铃形 | 高模块度、低跨模块边、两三个密集子群 | "阵营对立" — 蜀 vs 魏，忠 vs 奸 |
| **爱情戏** | 链式/星链状 | 低直径、Top-2节点占绝大多数边 | "才子佳人" — 男女主 + 功能性配角 |
| **侠义戏** | 英雄单核型 | 单一高中心性节点、外围松散 | "英雄入江湖" — 主角闯入陌生世界 |
| **神话戏** | 层级式/变体 | 高模块度但跨模块边也多 | "天界-凡间-地府" 三个世界的角色偶联 |
| **技法展示戏** | 极简/无结构 | 极少节点和边 | 独角戏或二人对戏，展示唱腔或做工 |

## 二、数据分析 Pipeline

### Phase 1：批量共现网络提取（全 1473 本）

**输入**：每本剧本 JSON 中的"正文对话"字段

**方法**（纯正则，无 LLM）：

```
1. 场景切分: re.split(r'【[^】]*(?:场|折|幕)[^】]*】', dialogue)
2. 每场提取角色: re.findall(r'^([\u4e00-\u9fa5]{2,4})\s+（', scene_text)
3. 共现边: 同场任意两角色间建立边，权重 = 共现场次数
```

**实现**：[p2_extract_networks.py](HumanVIZ/scripts/p2_extract_networks.py)

**产出**：`p2_networks.json` — 每本的 nodes（角色+度中心性+出场次数）+ edges（共现权重）+ 网络概要

### Phase 2：网络指标计算 + 统计检验

对每本剧本的网络计算 8 个结构指标：

| 指标 | 公式/含义 | 对类型的区分力 |
|------|---------|--------------|
| **密度 (density)** | 实际边数 / 最大可能边数 | 区分家庭戏(高) vs 侠义戏(低) |
| **中心性偏离度 (centralization)** | max_degree / mean_degree | 区分"单核" vs "多核"结构 |
| **聚类系数 (clustering)** | 邻居之间也相连的平均比例 | 区分公案戏(高) vs 爱情戏(低) |
| **模块度 (modularity)** | 标签传播社区检测的 Q 值 | 区分历史戏(高) vs 技法展示(低) |
| **有效直径 (diameter)** | Floyd-Warshall 90%百分位最短路径 | 区分大型戏 vs 小型戏 |
| **度分布熵 (degree_entropy)** | Shannon 熵归一化，衡量角色重要性的均匀度 | 区分家庭戏(高) vs 爱情戏(低) |
| **桥接节点比 (bridge_ratio)** | 跨社区枢纽节点占比 | 区分侠义戏(高) vs 技法展示(低) |
| **Top-2集中度 (top2_concentration)** | 最强两边的权重占比 | 区分技法展示(高) vs 历史戏(低) |

**统计检验**：ANOVA + Kruskal-Wallis + Tukey HSD 事后比较

**实现**：[p2_analyze_networks.py](HumanVIZ/scripts/p2_analyze_networks.py)

**产出**：`p2_metrics.json`（1473条×8指标）+ `p2_type_comparison.json`（类型间显著性结果）

### Phase 3：前端数据生成

对 Phase 2 的结果做降维、采样、提取代表性网络，产出前端可用的紧凑 JSON。

**实现**：内联 Python 脚本

**产出**：[p2_frontend_data.json](HumanVIZ/src/data/p2_frontend_data.json)
- `type_means`：7 种类型的 8 指标均值
- `pca_points`：500 本采样的 PCA 坐标
- `pca_centroids`：7 种类型的 PCA 质心
- `rep_networks`：每类型 2 个代表性网络的 nodes + edges（用于力导向图）
- `key_findings`：每指标的极值类型 + 显著差异对
- `top_chars`：每类型枢纽度 Top-3 角色

## 三、关键发现

全部 8 项指标在 7 种类型间均达到 **p < 0.001** 显著水平。部分推翻了最初假设：

| 指标 | 最高类型 | 最低类型 | 解读 |
|------|---------|---------|------|
| **密度** | 技法展示 (0.896) | 侠义戏 (0.512) | 小网络天然高密度；侠义戏角色多而分散 |
| **中心性偏离度** | 侠义戏 (1.246) | 技法展示 (0.102) | ✓ 验证英雄单核假设 |
| **聚类系数** | 公案戏 (0.855) | 爱情戏 (0.744) | ✗ 推翻星形假设：公案戏是"密集核心+辐射散边" |
| **模块度** | 侠义戏 (0.122) | 技法展示 (0.000) | 侠客连接不同世界（官府与江湖） |
| **度分布熵** | 家庭戏 (0.956) | 爱情戏 (0.911) | ✓ 验证家庭戏"多核心扁平"结构 |
| **桥接节点比** | 侠义戏 (0.069) | 技法展示 (0.000) | 英雄在多个社区间架桥 |
| **Top-2集中度** | 技法展示 (0.592) | 历史戏 (0.148) | 二人对戏集中；群像戏散焦 |
| **角色数** | 侠义戏 (17.4) | 技法展示 (3.5) | 侠义戏角色最多，技法展示极少 |

### 修正后的类型网络画像

| 剧目类型 | 修正后的网络画像 |
|---------|----------------|
| **公案戏** | 密集核心（衙役/官吏团） + 辐射散边（当事人/证人），聚类系数最高 |
| **家庭戏** | 扁平团块，角色权重最均匀，度熵最高 |
| **侠义戏** | 英雄单核 + 多社区桥接 + 低密度，中心性偏离度最高 |
| **历史戏** | 群像散焦 + 模块化阵营 + Top-2集中度最低 |
| **爱情戏** | 双核链式，中庸各项指标 |
| **神话戏** | 中等规模，各项指标居中 |
| **技法展示戏** | 极简网络，2-4人，密度极高 |

## 七、与任务四的协同

| 协同维度 | 任务二输出 | 任务四可复用 |
|---------|-----------|------------|
| 场景切分正则 | 自适应 `【场/折/幕】` 模式 | 同一正则（已共享） |

| 剧目类型分组 | 7 类型标签（全量 1473 本） | 叙事结构 × 类型交叉分析 |
| 网络密度时序 | 每场景的累计边数增长曲线 | 可与叙事节奏曲线并排对比 |
| 典型剧本选取 | 每类型的网络结构代表性 | 可作为任务四典型剧本的交叉验证 |
