# ChinaVis 2026 赛道1-I
# 任务三《剧本主题提取与跨剧本比较》设计流程指导报告

## 一、任务目标解析

任务三的核心目标是：基于全量京剧剧本的情节摘要文本，构建主题分类体系，提取每本剧本的主题向量，并比较不同剧目类型之间在主题内容上的系统差异，揭示各类型的"主题指纹"与跨剧本的主题共现模式。

该任务本质上融合了：

- 文本挖掘（Text Mining）
- 关键词匹配与加权计分
- 统计检验（卡方独立性检验）
- 主题共现网络分析
- 可视分析

因此，最终成果应体现：

| 层次 | 内容 |
|------|------|
| 数据层 | 12 维主题关键词体系构建与情节摘要匹配 |
| 算法层 | 加权计分 + 归一化的主题向量提取 |
| 分析层 | 剧目类型×主题覆盖率的统计检验与差异分析 |
| 洞察层 | 七种剧目类型的"主题指纹"画像与主题共现规律 |
| 可视层 | 主题关联环图、雷达图、热力图等多视图交互系统 |

## 二、整体研究框架

建议整体流程如下：

```
情节摘要文本（1473本）
  ↓
12 维主题关键词体系构建
  ↓
关键词匹配与加权计分
  ↓
主题向量提取（归一化 + 二值呈现）
  ↓
统计检验：卡方独立性检验 (χ²)
  ↓
剧目类型×主题差异分析
  ↓
主题共现模式挖掘
  ↓
可视分析与交互展示
```

### 3.2 主题提取方法

采用纯关键词匹配方案（无 LLM 依赖），保证处理效率与可复现性：

| 步骤 | 方法 | 说明 |
|------|------|------|
| 文本输入 | 每本剧本 JSON 中的"情节"字段 | 1467 本有情节文本 |
| 关键词匹配 | 逐一匹配 12 个主题的关键词列表 | 长关键词权重更高 |
| 加权计分 | `score = Σ(count(kw) × len(kw))` | 长关键词权重更高 |
| 归一化 | 将 12 维得分向量归一化到 [0,1] | 便于跨剧本比较 |
| 呈现判定 | score > 0 则认为该主题存在 | 二值化主题呈现 |

## 四、主题分析模型设计

该阶段是整个任务的核心分析基础。

### 4.1 统计检验设计

不建议仅凭覆盖率比较下结论。采用 **"卡方独立性检验 + 差异分析 + 主题丰度 + 共现分析"多重验证方案**：

| 检验方法 | 用途 | 说明 |
|----------|------|------|
| 卡方独立性检验 (χ²) | 检验每个主题在 7 种类型间的分布是否独立 | p < 0.01 为显著 |
| 差异分析 (Cohen-like) | 类型的主题覆盖率 − 全局主题覆盖率 | 量化类型的强势/弱势主题 |
| 主题丰度 | 每类型平均主题数 | 衡量类型的叙事复杂度 |
| 共现分析 | 主题对在同一剧本中的共现频次 | 揭示主题组合规律 |

该方案的优势在于：多重检验交叉验证，避免单一方法的假阳性风险，确保结论的统计学可信度。

### 4.2 主题向量结构

每本剧本的主题向量包含三个层次：

```
剧本主题向量
├── 原始得分（raw score）—— 各主题的加权匹配得分
├── 归一化得分（normalized）—— [0,1] 区间，便于跨剧本比较
└── 二值呈现（binary）—— 该主题是否存在于该剧本
```

## 七、交互系统设计建议

采用三面板布局实现多视图联动。

**布局设计**：

| 区域 | 内容 |
|------|------|
| 左侧 | 剧目类型选择 + 区分主题排名 + 主题丰度统计 |
| 中间 | 主视图切换区（主题共现热力图 / 类型主题雷达 / 主题组合分布） |
| 右侧 | 主题覆盖率排名（12 主题全局覆盖率 + 进度条）+ 高频主题共现对 |

**交互逻辑**：

- 点击左侧剧目类型 → 中间主视图切换至该类型的主题分布
- 切换主视图模式 → 在热力图、雷达图、组合分布图间自由切换
- 右侧抽屉面板 → 主题提取方法 / 组合模式详情 / 统计检验 三选一

**推荐设计语言**：

采用与任务一、二统一的"燕京清晖"主题——木色/卡其/竹青/纸白为主色调，玻璃态面板（双层渐变+半透明层+圆角+阴影），PT Serif / Noto Serif SC 字体，hover 浮动 + active 缩放的交互反馈。

## 八、论文/答卷中的创新点包装

建议写法：

- **创新点1**：构建面向京剧剧本的 12 维主题分类体系，融合戏曲学领域知识与文本挖掘方法
- **创新点2**：提出基于加权关键词匹配的主题向量提取方法，长关键词自适应加权提升匹配精度
- **创新点3**：通过卡方检验量化七种剧目类型的"主题指纹"，揭示主题与类型的系统关联
- **创新点4**：发现京剧"宫廷+家族"双核叙事空间等主题共现规律，为戏曲叙事学提供计算证据

## 九、推荐技术栈

- **数据处理**：Python（批量关键词匹配）、NumPy（数值计算）、pandas（结构化分析）
- **统计检验**：SciPy（卡方检验）、statsmodels（多重比较校正）
- **可视化**：ECharts（热力图、雷达图、环图、树图、散点图）、D3.js（自定义关联图）
- **前端框架**：React + TypeScript + SCSS（组件化开发）

## 十、系统实现与成果展示

最终呈现为 **"戏曲剧本主题提取与跨剧本比较可视分析系统"**。核心强调：**可量化 · 可比较 · 有主题洞察 · 有统计支撑**。

实际系统的关键可视化组件：

| 组件 | 实现方式 | 功能 |
|------|----------|------|
| **主题共现热力图** | ECharts heatmap | 7类型×12主题覆盖率矩阵，聚类排列，颜色编码覆盖率 |
| **类型主题雷达图** | ECharts radar | 12 轴雷达图展示各类型的多维主题画像 |
| **主题组合分布图** | ECharts bar | 展示独特主题组合的频次分布与类型交叉 |
| **抽屉面板** | React 组件 | 主题提取方法 / 组合模式详情 / 统计检验 三个分析面板 |
| **左侧类型选择** | React 按钮组 | 7 种剧目类型切换，联动主视图更新 |

系统采用前后端分离架构：前端 React + TypeScript + ECharts，数据预处理 Python 脚本批量生成静态 JSON，支持 1473 本剧本的全量主题分析与交互探索。

## 十一、与任务一、二的协同设计

任务三与任务一、二在数据管道和分析维度上形成有机协同：

| 协同维度 | 任务一（角色行当） | 任务二（角色关系） | 任务三（主题结构） |
|----------|-------------------|-------------------|-------------------|
| 核心对象 | 单个角色的属性 | 角色间的共现关系 | 剧本的主题内容 |
| 分析粒度 | 角色级 | 网络级 | 剧本级 |
| 文本来源 | 台词与舞台提示 | 场景级角色共现 | 情节摘要 |
| 可视化重心 | Sankey 流图 | 力导向图 | 主题关联环图 |
| 分类对比 | 雷达图 | 指标热力图 | 主题热力图 |
| 共享资源 | 角色特征 Schema | 剧目类型标签 | 剧目类型标签 |

三者共同构成从"角色个体分析"到"角色关系分析"再到"剧本主题分析"的完整戏曲数字人文研究链路。
