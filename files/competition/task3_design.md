## 一、核心分析问题

## 二、数据分析 Pipeline

### Phase 1：关键词主题提取（全 1473 本）

**输入**：每本剧本 JSON 中的"情节"（情节概要）字段（1467 本有情节文本）

**方法**（纯关键词匹配，无 LLM）：

```
1. 对每本的情节文本，逐一匹配 12 个主题的关键词列表
2. 加权计分：score = Σ(count(kw) × len(kw))  — 长关键词权重更高
3. 归一化：将 12 维得分向量归一化到 [0,1]
4. 主题呈现判定：score > 0 则认为该主题存在于该剧本
```

**实现**：[p3_extract_themes.py](HumanVIZ/scripts/p3_extract_themes.py)

**产出**：`p3_themes.json`
- 每本：12 维主题向量（原始得分 + 归一化 + 二值呈现）
- 匹配的关键词列表
- 12 个主题的全局统计
- 类型-主题交叉统计
- 前 20 对主题共现

### Phase 2：主题分布分析 + 跨类型比较

**统计检验**：

| 方法 | 用途 |
|------|------|
| 卡方独立性检验 (χ²) | 每个主题在 7 种类型间的分布是否独立 |
| 差异分析 (Cohen-like) | 类型的主题覆盖率 − 全局主题覆盖率 |
| 主题丰度 | 每类型平均主题数 |
| 共现分析 | 主题对在同一剧本中的共现频次 |

**实现**：[p3_analyze_themes.py](HumanVIZ/scripts/p3_analyze_themes.py)

**产出**：`p3_type_comparison.json`
- 7 类型 × 12 主题的覆盖率矩阵
- 每类型的区分性主题排名
- 卡方检验结果

|---------|---------------------|-----------|

| **家庭戏** | 家庭伦理 > 爱情姻缘 > 科举功名 | +26%, +14%, +6% |
| **侠义戏** | 侠义江湖 > 生死离别 > 爱情姻缘 | +30%, +11%, +9% |
| **爱情戏** | 爱情姻缘 > 科举功名 > 神话灵异 | +23%, +5%, +3% |
| **神话戏** | 神话灵异 > 侠义江湖 > 宫廷朝堂 | +64%, +9%, +7% |
| **公案戏** | 冤案昭雪 > 智谋韬略 > 权谋斗争 | +45%, +22%, +19% |

### 4.2 四张可视化图表

**图 1 — 主题关联环图**（主视图，ECharts circular graph）

- 12 个节点按圆形排列（每个节点=一个主题，颜色编码）
- 节点大小 = 主题全局覆盖率
- 边 = 主题共现关系（Top 15 对，边宽 ∝ 共现频次）
- 可拖拽、hover 显示共现数
- 等效于简化的 Chord Diagram

**图 2 — 主题树图**（备选视图，ECharts treemap）

- 按所选剧目类型的主题覆盖率分配面积
- 颜色 = 主题编码色
- 大小 = 覆盖率，标签显示主题名
- 直观展示类型的"主题投资组合"

**图 3 — 类型主题画像散点图**（备选视图，ECharts scatter）

- X 轴 = 全局主题覆盖率，Y 轴 = 所选类型的主题覆盖率
- 虚线 y=x 为参考线（上方=高于全局，下方=低于全局）
- 气泡大小 = 差异绝对值
- 标注主题名
- 一眼区分"该类型的强势主题"和"弱势主题"

**图 4 — 横向堆叠条**（页脚，ECharts bar）

- Y 轴 = 12 个主题，X 轴 = 覆盖率
- 6 种剧目类型堆叠
- 颜色 = 类型编码色

## 七、局限与改进方向

1. **关键词方法的粒度**：当前仅区分 12 个大类，未细分子主题（如"征战"可分为"防御战/征讨战/追击战"等）
2. **隐式主题**：反讽、哲学思辨等需要深层文本理解的抽象主题无法用关键词捕获，需 LLM 补充
3. **情节摘要的偏倚**：摘要由不同编者撰写，详略不一（部分仅 30 字），可能影响主题完整性
4. **技法展示戏样本过少**：仅 15 本，统计稳定性不足
