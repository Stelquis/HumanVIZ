# ChinaVis 2026 赛道1-I
# 任务四《叙事结构分析与模式总结》设计流程指导报告

## 一、任务目标解析

任务四的核心目标是：基于全量 1473 本京剧剧本的结构化特征，提取叙事结构指纹，通过分层抽样选取典型剧本进行深度叙事分析，并基于"Story Ribbons"可视化范式改造，构建面向京剧剧本的叙事结构可视分析系统。

该任务本质上融合了：

- 叙事学（Narratology）
- 结构特征提取与聚类分析
- LLM 驱动的深度语义标注
- 可视化叙事分析（Story Ribbons 改造）
- 分层抽样与覆盖性验证

因此，最终成果应体现：

| 层次 | 内容 |
|------|------|
| 数据层 | 1473 本剧本的 30 维结构指纹批量提取 |
| 算法层 | 层次聚类 + 分层抽样 + LLM 深度标注 |
| 分析层 | 叙事模式识别与跨类型/跨时代比较 |
| 洞察层 | 典型叙事结构模式归纳 |
| 可视层 | 三面板交互式叙事结构分析系统 |

## 二、整体研究框架

建议整体流程如下：

```
剧本文本（1473本）
  ↓
Phase 1: 批量结构指纹提取（纯正则，30维特征）
  ↓
Phase 2: 层次聚类 + 叙事结构类型划分
  ↓
Phase 3: 分层抽样（剧目类型×来源时代×结构类型）
  ↓
Phase 4: LLM 深度叙事分析（40-60本典型剧本）
  ↓
Phase 5: Story Ribbons 改造 + 三面板可视分析系统
```

## 三、数据预处理阶段

### 3.1 剧本结构解析

京剧剧本正文具备天然的结构化标记，为叙事结构分析提供了精确的解析基础：

| 结构信号 | 格式示例 | 叙事含义 |
|----------|----------|----------|
| 场景标记 | `【第X场】` / `【第X折】` | 天然的叙事结构边界 |
| 表演类型 | `（唱）/（念）/（白）/（做）/（打）` | 叙事模式的直接指标 |
| 唱腔板式 | `西皮摇板/二黄慢板/西皮快板` | 情绪强度信号（快板→紧张，慢板→抒情） |
| 舞台指示 | `（急急风）/（四击头）/（哭）/（笑）` | 剧情情绪标记 |

### 3.2 结构指纹提取

采用纯正则方案对全量 1473 本剧本提取约 30 维结构特征向量：

| 维度类别 | 具体特征 | 叙事结构含义 |
|----------|----------|------------|
| **场景规模** | 场次数、总行数、平均每场行数、场景行数变异系数 | 叙事复杂度与节奏均匀度 |
| **表演类型分布** | 唱/念/白/做/打类占比 | 叙事模式（唱工戏/做工戏/文武兼备） |
| **唱腔细分** | 西皮/二黄占比、板式多样性、快板/慢板比 | 情绪表达丰富度与张力 |
| **场景节奏** | 场景间行数变化率、首末场长度比、最长场位置 | 节奏起伏与高潮位置 |
| **情绪标记** | 笑/哭频次、冲突类指示密度、锣鼓经密度 | 情绪基调（喜剧/正剧/悲剧） |
| **角色密度** | 总角色数、平均每场角色数、Top-3集中度 | 叙事聚焦类型（群像/单人） |

### 3.3 三维分类体系

不依赖单一分类维度，使用"剧目类型 × 来源时代 × 叙事结构聚类"三个正交维度构建分层网格：

**维度 A — 剧目类型（7 类）**：历史戏、家庭戏、侠义戏、爱情戏、神话戏、公案戏、技法展示戏

**维度 B — 来源与时代语境（5 类）**：民国汇编本（1910s-1940s）、新中国整理本（1950s-1980s）、名家演出本、昆曲及其他、现代剧作家本

**维度 C — 叙事结构类型（聚类得出）**：密集高潮型（2-4场）、长篇铺陈型（10+场）、文武交替型（5-12场）、渐进推进型（5-9场）、无场景标记型

## 四、典型剧本选取策略

### 4.1 分层抽样设计

采用四步分层抽样策略，确保全方位覆盖：

| 步骤 | 方法 | 说明 |
|------|------|------|
| Step 1 | 构建 7×5 分层网格 | 35 个格单元，标注每格剧本数量 |
| Step 2 | 格内子聚类 + 重心距离 | 每个子类选离聚类中心最近的 1 本为"典型代表" |
| Step 3 | 特殊覆盖规则 | 无场景标记型≥3本、昆曲≥3本、技法展示戏全覆盖 |
| Step 4 | 校验与调整 | 确保 7 类型×5 来源×5 结构类型全覆盖 |

### 4.2 覆盖性验证

| 覆盖维度 | 是否覆盖 | 说明 |
|----------|----------|------|
| 7 种剧目类型 | ✓ | 每类至少 2-3 本 |
| 5 种来源语境 | ✓ | 每类有代表 |
| 所有叙事结构聚类 | ✓ | 每类有代表 |
| 无场景标记特例 | ✓ | 至少 3 本（占 21.5%） |
| 昆曲非京剧样本 | ✓ | 至少 3 本 |
| 13 位名家流派 | ✓ | 每家至少 1 本 |

## 五、深度叙事分析设计

### 5.1 三层分析框架

对 40-60 本典型剧本进行三层深度叙事分析：

**Layer 1 — 场景级节奏标注**：每场标注表演形式配比（5维）、唱腔板式序列、情绪强度（1-10分）、冲突级别（1-10分）、叙事功能（起/承/转/合四分类）。

**Layer 2 — 全局叙事模式识别**：每本识别叙事弧线类型（峰型/坡型/谷型/双峰型/平缓型）、阶段边界（起承转合四阶段占比）、节奏模式（紧-松-紧/松-紧-松/匀速递进/爆发式）、收束方式（团圆/开放/悲壮/教化）。

**Layer 3 — 表演形式-叙事功能映射**：建立唱念做打各表演形式与叙事功能的对应关系框架。

### 5.2 表演形式的叙事功能

| 表演形式 | 叙事功能 | 典型场景 |
|----------|----------|----------|
| 唱（慢板/原板） | 抒情独白、内心活动外化 | 月下思人、狱中诉冤 |
| 唱（快板/摇板） | 情绪爆发、冲突升级 | 闻报惊变、怒斥奸佞 |
| 念（引子/定场诗） | 角色登场亮相、身份建立 | 每场开场 |
| 白 | 日常对话、喜剧互动、剧情推演 | 贯穿全剧 |
| 打 | 冲突高潮、正邪对决 | 战场、搏斗 |

## 六、可视化设计方案

### 6.1 Story Ribbons 编码改造

将原始 Story Ribbons（英文小说角色轨迹）改造为京剧叙事 Ribbon 编码：

| 编码维度 | 原始方案 | 京剧改造方案 |
|----------|----------|------------|
| X 轴 | 故事进度 | 场景序列（归一化为 0%-100% 进度） |
| Y 轴 | 角色名称 | 三层叠加：情绪曲线 + 表演形式色带 + 冲突热力 |
| 颜色 | 角色情绪 | 表演形式（5色离散：唱红/念蓝/做绿/打橙/白灰） |
| 宽度 | 角色重要性 | 对话密度 / 多角色同场程度 |
| 背景 | 无 | 起承转合四阶段（4色半透明） |

### 6.2 三面板布局

| 面板 | 内容 | 用途 |
|------|------|------|
| **面板 1：叙事结构对比总览** | 多本 Ribbon 并排展示，按类型/时代/结构分组 | 横向比较不同剧本的叙事结构差异 |
| **面板 2：单本深度丝带图** | 完整 Ribbon + 角色出场热力图 + 板式分布 | 深入分析单本剧本的叙事结构细节 |
| **面板 3：叙事模式聚类空间** | PCA/t-SNE 散点图（1473本） | 探索叙事结构的聚类模式 |

### 6.3 交互设计

| 交互 | 触发方式 | 效果 |
|------|----------|------|
| 场景详情 | hover 场景块 | Tooltip: 表演形式、角色、情绪值、原文片段 |
| 进入单本视图 | click Ribbon | 切换到面板 2，展示完整叙事结构 |
| 区域筛选 | brush 散点面板 | 筛选同结构类型的剧本 |
| 切换对比模式 | 下拉选择 | 按剧目类型/来源/叙事结构类型重新分组 |
| 原文联动 | click 场景块 | 侧边栏展开该场完整对话文本 |

## 七、论文/答卷中的创新点包装

建议写法：

- **创新点1**：提出面向京剧剧本的 30 维结构指纹提取方法，纯正则实现 1473 本全覆盖
- **创新点2**：构建"剧目类型×来源时代×叙事结构"三维正交分层抽样框架，确保典型剧本的全方位覆盖
- **创新点3**：将 Story Ribbons 可视化范式改造为京剧叙事分析工具，以表演形式色带替代角色轨迹为核心编码
- **创新点4**：建立表演形式-叙事功能映射分析框架，为京剧叙事学提供计算分析视角

## 八、推荐技术栈

- **数据处理**：Python（正则批量提取）、pandas（结构化分析）、NumPy（数值计算）
- **聚类分析**：scikit-learn（层次聚类、PCA）、SciPy（轮廓系数）
- **深度标注**：LangChain + Pydantic（LLM 异步并发标注）
- **可视化**：D3.js（Story Ribbons 核心渲染）、ECharts（辅助图表）、React + TypeScript + Zustand（前端框架）

## 九、系统实现与成果展示

最终呈现为 **"京剧叙事结构可视分析系统"**。核心强调：**可比较 · 可探索 · 有叙事洞察 · 有结构支撑**。

实际系统的关键可视化组件：

| 组件 | 实现方式 | 功能 |
|------|----------|------|
| **叙事 Ribbon 图** | D3.js + React | 多层叠加的叙事结构可视化（表演形式色带+情绪曲线+冲突热力） |
| **叙事结构对比总览** | D3.js 布局 | 多本 Ribbon 并排，按类型/时代/结构分组排列 |
| **单本深度丝带图** | D3.js + ECharts | 完整 Ribbon + 角色出场热力图 + 唱腔板式分布 |
| **叙事模式聚类空间** | ECharts scatter | PCA/t-SNE 散点图，颜色=类型，点击联动 |
| **叙事模式归纳面板** | React 组件 | 基于全部数据的典型叙事模式总结与可视化 |

系统采用前后端分离架构：前端 React + TypeScript + D3.js + ECharts，数据预处理 Python 脚本批量生成 JSON，LLM Pipeline 异步并发深度标注，支持 1473 本剧本的全量叙事结构分析与交互探索。

## 十、与任务一、二、三的协同设计

任务四与前三个任务在数据管道和分析维度上形成完整闭环：

| 协同维度 | 任务一（行当） | 任务二（关系） | 任务三（主题） | 任务四（叙事） |
|----------|---------------|---------------|---------------|---------------|
| 核心对象 | 角色属性 | 角色关系 | 剧本主题 | 叙事结构 |
| 分析粒度 | 角色级 | 网络级 | 剧本级 | 场景级 |
| 文本来源 | 台词与提示 | 场景共现 | 情节摘要 | 正文对话 |
| 共享资源 | 角色行当 | 剧目类型标签 | 剧目类型标签 | 剧目类型+来源时代 |
| 可视化重心 | Sankey 流图 | 力导向图 | 热力图 | Story Ribbons |

四者共同构成从"角色个体"到"角色关系"到"剧本主题"再到"叙事结构"的完整戏曲数字人文研究链路，实现多维度、多层次的京剧剧本数字化分析体系。
